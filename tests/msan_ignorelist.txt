# The sqids-cpp blocklist header constructs/destructs global std::string objects
# during static initialization. With -fsanitize-memory-use-after-dtor, the
# destructor poisons those strings' memory. When the Azure CurlConnectionPool
# global constructor later calls curl_global_init -> OPENSSL_init_ssl, OpenSSL
# loads the uninstrumented FIPS provider (/etc/ssl/fips.so) which reads memory
# that overlaps with the poisoned region, triggering a false positive.
