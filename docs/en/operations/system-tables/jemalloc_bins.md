## Description {#description}

Contains information about memory allocations done via jemalloc allocator in different size classes (bins) aggregated from all arenas.
These statistics might not be absolutely accurate because of thread local caching in jemalloc.

## Columns {#columns}

<!--AUTOGENERATED_START-->
- `index` ([UInt16](/sql-reference/data-types/int-uint)) — Index of the bin ordered by size.
- `large` ([UInt8](/sql-reference/data-types/int-uint)) — True for large allocations and False for small.
- `size` ([UInt64](/sql-reference/data-types/int-uint)) — Size of allocations in this bin.
- `allocations` ([Int64](/sql-reference/data-types/int-uint)) — Number of allocations.
- `deallocations` ([Int64](/sql-reference/data-types/int-uint)) — Number of deallocations.
- `nregs` ([Int64](/sql-reference/data-types/int-uint)) — Number of regions per slab.
- `curslabs` ([Int64](/sql-reference/data-types/int-uint)) — Current number of slabs.
- `curregs` ([Int64](/sql-reference/data-types/int-uint)) — Current number of regions for this size class.
<!--AUTOGENERATED_END-->

**Aliases:**

- `availregs` — Alias for `nregs * curslabs`.
- `util` — Alias for `curregs / availregs`.

## Example {#example}

Find the sizes of allocations that contributed the most to the current overall memory usage.

```sql
SELECT
    *,
    allocations - deallocations AS active_allocations,
    size * active_allocations AS allocated_bytes
FROM system.jemalloc_bins
WHERE allocated_bytes > 0
ORDER BY allocated_bytes DESC
LIMIT 10
```
