## Description {#description}

Contains a list of all available failpoints registered in the server, along with their type and whether they are currently enabled.

Failpoints can be enabled and disabled at runtime using the `SYSTEM ENABLE FAILPOINT` and `SYSTEM DISABLE FAILPOINT` statements.

## Columns {#columns}

<!--AUTOGENERATED_START-->
- `name` ([String](/sql-reference/data-types/string)) ‚Äî Name of the failpoint.
- `type` ([Enum8](/sql-reference/data-types/enum)) ‚Äî Type of the failpoint. Possible values:
  - `'once'` ‚Äî Triggers a single time and then auto-disables.
  - `'regular'` ‚Äî Triggers every time the failpoint is hit.
  - `'pauseable_once'` ‚Äî Blocks execution once until explicitly resumed.
  - `'pauseable'` ‚Äî Blocks execution every time the failpoint is hit until explicitly resumed.
- `enabled` ([UInt8](/sql-reference/data-types/int-uint)) ‚Äî Whether the failpoint is currently enabled. `1` means enabled, `0` means disabled.
<!--AUTOGENERATED_END-->

## Example {#example}

```sql title="Query"
SYSTEM ENABLE FAILPOINT replicated_merge_tree_insert_retry_pause;
SELECT * FROM system.fail_points WHERE enabled = 1
```
