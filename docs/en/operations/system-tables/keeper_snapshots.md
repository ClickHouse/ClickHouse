## Description {#description}

This table does not exist if this node is not configured to run an in-process ClickHouse Keeper. It contains one row per Raft snapshot file tracked by the in-process Keeper state machine, including snapshots currently being received from the leader.

## Columns {#columns}

<!--AUTOGENERATED_START-->
-   `last_log_index` ([UInt64](../../sql-reference/data-types/int-uint.md)) — Last log index covered by the snapshot.
-   `path` ([String](../../sql-reference/data-types/string.md)) — Snapshot file path on the disk.
-   `disk_name` ([String](../../sql-reference/data-types/string.md)) — Name of the disk that stores the snapshot.
-   `size_bytes` ([UInt64](../../sql-reference/data-types/int-uint.md)) — Size of the snapshot file on disk.
-   `last_modified_at` ([DateTime](../../sql-reference/data-types/datetime.md)) — Last modification time of the snapshot file.
-   `is_received` ([Bool](../../sql-reference/data-types/boolean.md)) — `true` if the snapshot is currently being received from the leader. For such rows `size_bytes` and `last_modified_at` reflect the partial file as written so far and may underreport.
-   `exists_on_disk` ([Bool](../../sql-reference/data-types/boolean.md)) — Whether the snapshot file is currently present on disk. Always `true` for finalized snapshots (`is_received` = `false`) unless the file was removed out of band or is corrupted; may be `false` for snapshots being received (`is_received` = `true`) before any bytes are written.
<!--AUTOGENERATED_END-->

Example:

```sql
SELECT * FROM system.keeper_snapshots ORDER BY last_log_index;
```
