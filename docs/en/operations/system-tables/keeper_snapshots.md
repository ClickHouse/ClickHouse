## Description {#description}

This table does not exist if this node is not configured to run an in-process ClickHouse Keeper. It contains one row per Raft snapshot file tracked by the in-process Keeper state machine, including snapshots currently being received from the leader.

## Columns {#columns}

<!--AUTOGENERATED_START-->
-   `last_log_index` ([UInt64](../../sql-reference/data-types/int-uint.md)) вҖ” Last log index covered by the snapshot.
-   `path` ([String](../../sql-reference/data-types/string.md)) вҖ” Snapshot file path on the disk.
-   `disk_name` ([String](../../sql-reference/data-types/string.md)) вҖ” Name of the disk that stores the snapshot.
-   `size_bytes` ([UInt64](../../sql-reference/data-types/int-uint.md)) вҖ” Size of the snapshot file on disk.
-   `last_modified_at` ([DateTime](../../sql-reference/data-types/datetime.md)) вҖ” Last modification time of the snapshot file.
-   `is_received` ([Bool](../../sql-reference/data-types/boolean.md)) вҖ” `true` if the snapshot is currently being received from the leader. For such rows `size_bytes` and `last_modified_at` reflect the partial file as written so far and may underreport.
<!--AUTOGENERATED_END-->

Example:

```sql
SELECT * FROM system.keeper_snapshots ORDER BY last_log_index;
```
