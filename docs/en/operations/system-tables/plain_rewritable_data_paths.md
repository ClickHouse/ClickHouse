## Description {#description}

Contains the local-to-remote path mapping of every `plain_rewritable` disk, enumerated directly from the in-memory metadata tree.

Each row represents one blob object backing a file on a `plain_rewritable` disk.

Like [system.remote_data_paths](/operations/system-tables/remote_data_paths), this table maps local file paths to their remote blob paths in object storage. The difference is the disk `metadata_type`: `system.remote_data_paths` serves disks that keep per-file metadata on the local filesystem, whereas `plain_rewritable` disks keep their metadata in memory. This table reads that in-memory metadata tree directly.

:::note
This table reflects only the **in-memory** metadata state maintained by the current server, and only for `plain_rewritable` disks that are **currently attached** (loaded) on this server. Disks that are configured but were never initialized do not appear. The in-memory state is populated on load and updated on refresh — it is not a live listing of object storage, so it may lag behind recent changes and may list entries that have since been removed from object storage by a concurrent operation.
:::

Because the table exposes the internal object storage layout of every table on the disk without per-table access filtering, reading it requires the `SHOW TABLES` privilege (granted globally, on `*.*`).

## Columns {#columns}

<!--AUTOGENERATED_START-->
- `hostname` ([String](/sql-reference/data-types/string)) — Hostname of the server reporting the in-memory metadata state.
- `disk_name` ([String](/sql-reference/data-types/string)) — Name of the `plain_rewritable` disk.
- `common_prefix_for_blobs` ([String](/sql-reference/data-types/string)) — Object storage key prefix common to all blobs of this disk.
- `local_path` ([String](/sql-reference/data-types/string)) — Logical directory path the file belongs to.
- `directory_remote_path` ([String](/sql-reference/data-types/string)) — Randomly-generated object storage directory backing the logical directory.
- `name` ([String](/sql-reference/data-types/string)) — File name inside the directory.
- `remote_path` ([String](/sql-reference/data-types/string)) — Full blob object key in object storage.
- `size` ([UInt64](/sql-reference/data-types/int-uint)) — Size of the file in bytes.
- `last_modified` ([DateTime](/sql-reference/data-types/datetime)) — Last modification time of the blob.
- `is_ephemeral` ([UInt8](/sql-reference/data-types/int-uint)) — `1` when the blob looks like an ephemeral temporary object (a recursive-removal rename target or a move/unlink scratch copy) rather than regular table data. This also matches in-flight operations, so it does not by itself mean a leak; an entry that persists across refreshes is a leak, an in-flight one disappears once the operation completes.
<!--AUTOGENERATED_END-->
