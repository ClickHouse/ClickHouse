## Description {#description}

Contains the local-to-remote path mapping of every `plain_rewritable` disk, enumerated directly from the in-memory metadata tree.

Each row represents one blob object backing a file on a `plain_rewritable` disk.

Unlike [system.remote_data_paths](/operations/system-tables/remote_data_paths), which walks the local `store`/`data`/`shadow` namespace, this table enumerates the metadata tree directly. As a result, it also surfaces directories that are not reachable through that namespace — for example, leftovers of an interrupted recursive removal, whose `local_path` is a randomly-generated name rather than a `store/...` path.

## Columns {#columns}

<!--AUTOGENERATED_START-->
- `disk_name` ([String](/sql-reference/data-types/string)) — Name of the `plain_rewritable` disk.
- `common_prefix_for_blobs` ([String](/sql-reference/data-types/string)) — Object storage key prefix common to all blobs of this disk.
- `local_path` ([String](/sql-reference/data-types/string)) — Logical directory path the file belongs to.
- `directory_remote_path` ([String](/sql-reference/data-types/string)) — Randomly-generated object storage directory backing the logical directory.
- `name` ([String](/sql-reference/data-types/string)) — File name inside the directory.
- `remote_path` ([String](/sql-reference/data-types/string)) — Full blob object key in object storage.
- `size` ([UInt64](/sql-reference/data-types/int-uint)) — Size of the file in bytes.
- `last_modified` ([DateTime](/sql-reference/data-types/datetime)) — Last modification time of the blob.
<!--AUTOGENERATED_END-->

```sql
SELECT * FROM system.plain_rewritable_data_paths LIMIT 1 FORMAT Vertical;
```
