## Description {#description}

Contains information about existing constraints in all tables.

Columns from [temporary tables](../../sql-reference/statements/create/table.md#temporary-tables) are visible in `system.constraints` only in the session where they were created. They are shown with an empty `database` field.

## Columns {#columns}

<!--AUTOGENERATED_START-->
- `database` ([String](/sql-reference/data-types/string)) — Database name.
- `table` ([String](/sql-reference/data-types/string)) — Table name.
- `name` ([String](/sql-reference/data-types/string)) — Constraint name.
- `type` ([Enum8('CHECK' = 0, 'ASSUME' = 1)](/sql-reference/data-types/enum)) — Constraint type.
- `expression` ([String](/sql-reference/data-types/string)) — Constraint expression.
<!--AUTOGENERATED_END-->

## Example {#example}

```sql
SELECT * FROM system.constraints LIMIT 2 FORMAT Vertical;
```

```text
Row 1:
──────
database:    default
table:       hits
name:        check_hits
type:        CHECK
expression:  CounterID > 0

Row 2:
──────
database:    default
table:       hits
name:        assume_positive
type:        ASSUME
expression:  WatchID > 0
```
