# 2026 Changelog

### ClickHouse release v26.2.8.15-stable (5f35e27d1a1) FIXME as compared to v26.2.7.17-stable (63ac8a8cc08)

#### Improvement
* Backported in [#101866](https://github.com/ClickHouse/ClickHouse/issues/101866): The text index is now GA and stays enabled regardless of the `compatibility` setting, preventing unexpected disabling during backup restores or when running in compatibility mode. [#101518](https://github.com/ClickHouse/ClickHouse/pull/101518) ([Nikita Fomichev](https://github.com/fm4v)).
* Backported in [#101860](https://github.com/ClickHouse/ClickHouse/issues/101860): Bump libarchive from 3.8.5 to 3.8.6. [#101658](https://github.com/ClickHouse/ClickHouse/pull/101658) ([Robert Schulze](https://github.com/rschu1ze)).
* Backported in [#101854](https://github.com/ClickHouse/ClickHouse/issues/101854): Bump curl from 8.18 to 8.19. [#101662](https://github.com/ClickHouse/ClickHouse/pull/101662) ([Robert Schulze](https://github.com/rschu1ze)).

#### Bug Fix (user-visible misbehavior in an official stable release)
* Backported in [#100217](https://github.com/ClickHouse/ClickHouse/issues/100217): Fixes a crash where we could de-reference a null pointer in system tables created between the time when we snapshot the tables in the `IDatabaseTablesIterator::table()` call and the tables changing in another thread during later iteration. [#98792](https://github.com/ClickHouse/ClickHouse/pull/98792) ([Grant Holly](https://github.com/grantholly-clickhouse)).
* Backported in [#100855](https://github.com/ClickHouse/ClickHouse/issues/100855): Fix heap-use-after-free when a table is dropped concurrently with a running read query (19 occurrences in CI over the last 90 days). [#99483](https://github.com/ClickHouse/ClickHouse/pull/99483) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Backported in [#101472](https://github.com/ClickHouse/ClickHouse/issues/101472): Fixed a bug in Keeper where a read request could get stuck (causing session to time out) if a different unrelated session on the same server was closed at just the wrong moment. [#99484](https://github.com/ClickHouse/ClickHouse/pull/99484) ([Michael Kolupaev](https://github.com/al13n321)).
* Backported in [#101393](https://github.com/ClickHouse/ClickHouse/issues/101393): Fix `session_timezone` being ignored when parsing `DateTime` values during async inserts (TCP) and all inserts over HTTP. [#100647](https://github.com/ClickHouse/ClickHouse/pull/100647) ([Sema Checherinda](https://github.com/CheSema)).
* Backported in [#101960](https://github.com/ClickHouse/ClickHouse/issues/101960): Fix wrong results when JOIN with shard-by-PK optimization uses query condition cache and some parts are filtered out by cached conditions. [#100926](https://github.com/ClickHouse/ClickHouse/pull/100926) ([Groene AI](https://github.com/groeneai)).
* Backported in [#101642](https://github.com/ClickHouse/ClickHouse/issues/101642): Fix undefined behavior (signed integer overflow) in parseDateTimeBestEffort when parsing datetime strings with more than 18 fractional-second digits. [#100948](https://github.com/ClickHouse/ClickHouse/pull/100948) ([Vasily Chekalkin](https://github.com/bacek)).
* Backported in [#101362](https://github.com/ClickHouse/ClickHouse/issues/101362): Fix `MAX`/`MIN` aggregate functions on `Decimal` columns returning incorrect results when JIT compilation is enabled (after the compilation threshold is reached). [#101203](https://github.com/ClickHouse/ClickHouse/pull/101203) ([Raúl Marín](https://github.com/Algunenano)).
* Backported in [#101371](https://github.com/ClickHouse/ClickHouse/issues/101371): ... [#101230](https://github.com/ClickHouse/ClickHouse/pull/101230) ([Kseniia Sumarokova](https://github.com/kssenii)).
* Backported in [#101561](https://github.com/ClickHouse/ClickHouse/issues/101561): Fix insert-select from delta lake cluster with replicated merge tree. [#101299](https://github.com/ClickHouse/ClickHouse/pull/101299) ([Konstantin Vedernikov](https://github.com/scanhex12)).
