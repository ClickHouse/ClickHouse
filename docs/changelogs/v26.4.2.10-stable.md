# 2026 Changelog

### ClickHouse release v26.4.2.10-stable (184f682d431) FIXME as compared to v26.4.1.1141-stable (7156188a3ba)

#### Performance Improvement
* Backported in [#104160](https://github.com/ClickHouse/ClickHouse/issues/104160): Avoid index uncompressed cache overhead when the cache is disabled (server setting `index_uncompressed_cache_size = 0`, which is the default). [#104063](https://github.com/ClickHouse/ClickHouse/pull/104063) ([Michael Kolupaev](https://github.com/al13n321)).

#### Improvement
* Backported in [#104213](https://github.com/ClickHouse/ClickHouse/issues/104213): Added SQL functions `obfuscateQuery` and `obfuscateQueryWithSeed` that obfuscate a SQL query string by replacing identifiers and literals with randomized placeholders while preserving the query structure. Fixed edge cases in query obfuscation for short identifiers and oversized integer literals. [#101027](https://github.com/ClickHouse/ClickHouse/pull/101027) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Backported in [#104224](https://github.com/ClickHouse/ClickHouse/issues/104224): Fix a memory leak when running queries with `MATERIALIZED` CTEs. [#104153](https://github.com/ClickHouse/ClickHouse/pull/104153) ([Alexey Milovidov](https://github.com/alexey-milovidov)).

#### Bug Fix (user-visible misbehavior in an official stable release)
* Backported in [#104211](https://github.com/ClickHouse/ClickHouse/issues/104211): Fix undefined behavior in `MergeTreeDataPartWriterCompact::cancel` when a stream allocation fails. [#101292](https://github.com/ClickHouse/ClickHouse/pull/101292) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Backported in [#104151](https://github.com/ClickHouse/ClickHouse/issues/104151): Disable trivial LIMIT optimization with row policies/additional_table_filters (to allow parallel index analysis). [#102921](https://github.com/ClickHouse/ClickHouse/pull/102921) ([Azat Khuzhin](https://github.com/azat)).
* Backported in [#104244](https://github.com/ClickHouse/ClickHouse/issues/104244): Fixed a data-correctness regression in 26.x where JIT-compiled `if` and `multiIf` with a `Decimal` result type and a non-`Decimal` (integer or float) literal in one branch silently returned a value `10^scale` too small. The slow (non-JIT) path was unaffected. Workaround on affected versions: `SET compile_expressions = 0`. [#103809](https://github.com/ClickHouse/ClickHouse/pull/103809) ([Groene AI](https://github.com/groeneai)).
* Backported in [#104147](https://github.com/ClickHouse/ClickHouse/issues/104147): Fix `SIZES_OF_ARRAYS_DONT_MATCH` when reading `AggregateFunction(topK(N), String)` columns whose persisted `alpha_size` was grown by a pre-25.12 deserialize+serialize cycle. [#104002](https://github.com/ClickHouse/ClickHouse/pull/104002) ([Raúl Marín](https://github.com/Algunenano)).
* Backported in [#104195](https://github.com/ClickHouse/ClickHouse/issues/104195): Fix write out of bounds while deserializing `quantileTiming` state. [#104141](https://github.com/ClickHouse/ClickHouse/pull/104141) ([Alexey Milovidov](https://github.com/alexey-milovidov)).

#### NOT FOR CHANGELOG / INSIGNIFICANT

* Backported in [#104208](https://github.com/ClickHouse/ClickHouse/issues/104208): `obfuscateQuery` and `obfuscateQueryWithSeed` now preserve names of built-in functions, aggregate functions, table functions, formats, data types, storages, codecs and settings, matching the behaviour of `clickhouse-format --obfuscate`. [#103843](https://github.com/ClickHouse/ClickHouse/pull/103843) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
