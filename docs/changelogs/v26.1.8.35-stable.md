# 2026 Changelog

### ClickHouse release v26.1.8.35-stable (fafb25db9c4) FIXME as compared to v26.1.7.13-stable (c5bf58bd2ff)

#### New Feature
* Backported in [#102363](https://github.com/ClickHouse/ClickHouse/issues/102363): Add asynchronous metrics for kernel TCP receive and transmit buffer memory (`sk_rmem_alloc`, `sk_wmem_alloc`) of HTTP connection pool sockets, reported as p50/p75/p90/p95 percentiles and totals per connection group. [#100575](https://github.com/ClickHouse/ClickHouse/pull/100575) ([Sema Checherinda](https://github.com/CheSema)).

#### Improvement
* Backported in [#102311](https://github.com/ClickHouse/ClickHouse/issues/102311): Disable AI SQL generation (`??` command) in the embedded client (SSH and WebSocket protocols) to prevent access to the server's environment variables. [#100290](https://github.com/ClickHouse/ClickHouse/pull/100290) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Backported in [#101599](https://github.com/ClickHouse/ClickHouse/issues/101599): Cgroupv2 memory tracking now excludes `slab_reclaimable` from kernel memory, giving a more accurate measure of non-reclaimable memory usage. [#100901](https://github.com/ClickHouse/ClickHouse/pull/100901) ([Antonio Andelic](https://github.com/antonio2368)).
* Backported in [#102136](https://github.com/ClickHouse/ClickHouse/issues/102136): Bumped internal mongo-c-driver from 2.2.2 to 2.2.3. [#101655](https://github.com/ClickHouse/ClickHouse/pull/101655) ([Robert Schulze](https://github.com/rschu1ze)).
* Backported in [#101925](https://github.com/ClickHouse/ClickHouse/issues/101925): Bump libarchive from 3.8.5 to 3.8.6. [#101658](https://github.com/ClickHouse/ClickHouse/pull/101658) ([Robert Schulze](https://github.com/rschu1ze)).
* Backported in [#101852](https://github.com/ClickHouse/ClickHouse/issues/101852): Bump curl from 8.18 to 8.19. [#101662](https://github.com/ClickHouse/ClickHouse/pull/101662) ([Robert Schulze](https://github.com/rschu1ze)).

#### Bug Fix (user-visible misbehavior in an official stable release)
* Backported in [#97102](https://github.com/ClickHouse/ClickHouse/issues/97102): Add more checks for the validation of a setting which is used as a root metadata file. [#96754](https://github.com/ClickHouse/ClickHouse/pull/96754) ([Daniil Ivanik](https://github.com/divanik)).
* Backported in [#99874](https://github.com/ClickHouse/ClickHouse/issues/99874): Fix `optimize_skip_unused_shards` optimization with the analyzer in case `Distributed` storage is used in a `View`. [#98754](https://github.com/ClickHouse/ClickHouse/pull/98754) ([Nikolai Kochetov](https://github.com/KochetovNicolai)).
* Backported in [#99669](https://github.com/ClickHouse/ClickHouse/issues/99669): Fix heap-use-after-free in `INTERSECT`/`EXCEPT` when the query produces duplicate column names. [#99471](https://github.com/ClickHouse/ClickHouse/pull/99471) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Backported in [#100853](https://github.com/ClickHouse/ClickHouse/issues/100853): Fix heap-use-after-free when a table is dropped concurrently with a running read query (19 occurrences in CI over the last 90 days). [#99483](https://github.com/ClickHouse/ClickHouse/pull/99483) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Backported in [#101471](https://github.com/ClickHouse/ClickHouse/issues/101471): Fixed a bug in Keeper where a read request could get stuck (causing session to time out) if a different unrelated session on the same server was closed at just the wrong moment. [#99484](https://github.com/ClickHouse/ClickHouse/pull/99484) ([Michael Kolupaev](https://github.com/al13n321)).
* Backported in [#99897](https://github.com/ClickHouse/ClickHouse/issues/99897): Fix query:. [#99739](https://github.com/ClickHouse/ClickHouse/pull/99739) ([Konstantin Vedernikov](https://github.com/scanhex12)).
* Backported in [#102366](https://github.com/ClickHouse/ClickHouse/issues/102366): Fixes a bug where ClickHouse could skip files if the `Content-Length` header was missing in their HEAD request response (for example, because of decompressive transcoding in GCS). [#99971](https://github.com/ClickHouse/ClickHouse/pull/99971) ([Yarik Briukhovetskyi](https://github.com/yariks5s)).
* Backported in [#101530](https://github.com/ClickHouse/ClickHouse/issues/101530): Some catalogs can show some secrets in the `SETTINGS` section of `select * from system.databases` query result. This PR prevent such behavoir. [#100800](https://github.com/ClickHouse/ClickHouse/pull/100800) ([Konstantin Vedernikov](https://github.com/scanhex12)).
* Backported in [#101183](https://github.com/ClickHouse/ClickHouse/issues/101183): Fix null pointer dereference segfault when loading dictionaries during server shutdown. `Context::getUserDefinedSQLObjectsStorage` (dereferences `user_defined_sql_objects_storage`) is called by dictionary threads concurrently with the main thread calling `Context::shutdown` (sets `user_defined_sql_objects_storage` to null). We need to make sure we disable future updates in the dictionaries loader, kill the currently running dictionary queries and join the dictionary loading threads - all before running `Context::shutdown`. Similar to what we do with normal queries. [#100839](https://github.com/ClickHouse/ClickHouse/pull/100839) ([Miсhael Stetsyuk](https://github.com/mstetsyuk)).
* Backported in [#101087](https://github.com/ClickHouse/ClickHouse/issues/101087): Fix buffer overflow in `ULIDStringToDateTime` when input contains non-ASCII bytes. [#100843](https://github.com/ClickHouse/ClickHouse/pull/100843) ([Konstantin Bogdanov](https://github.com/thevar1able)).
* Backported in [#101011](https://github.com/ClickHouse/ClickHouse/issues/101011): Fix exception in `intDiv`/`intDivOrZero` on arrays of nullable tuples, e.g. `SELECT intDiv([divide((1, 2), ... AND NULL)], 2)`. [#100895](https://github.com/ClickHouse/ClickHouse/pull/100895) ([Raúl Marín](https://github.com/Algunenano)).
* Backported in [#101246](https://github.com/ClickHouse/ClickHouse/issues/101246): Evaluate engine arguments for `StorageAlias` before storing the definition, so that expressions like `currentDatabase()` are resolved to literals before being saved to the database. [#100902](https://github.com/ClickHouse/ClickHouse/pull/100902) ([Nikolay Degterinsky](https://github.com/evillique)).
* Backported in [#101958](https://github.com/ClickHouse/ClickHouse/issues/101958): Fix wrong results when JOIN with shard-by-PK optimization uses query condition cache and some parts are filtered out by cached conditions. [#100926](https://github.com/ClickHouse/ClickHouse/pull/100926) ([Groene AI](https://github.com/groeneai)).
* Backported in [#101630](https://github.com/ClickHouse/ClickHouse/issues/101630): Fix undefined behavior (signed integer overflow) in parseDateTimeBestEffort when parsing datetime strings with more than 18 fractional-second digits. * Backported in [#101076](https://github.com/ClickHouse/ClickHouse/issues/101076): Fix `sumCountOrDefault` aggregate function with one or more `Nullable` argument not being able to read older serialized states after introduction of `Nullable(Tuple)`. Closes [#100882](https://github.com/ClickHouse/ClickHouse/issues/100882). [#101021](https://github.com/ClickHouse/ClickHouse/pull/101021) ([Nihal Z. Miaji](https://github.com/nihalzp)).
* Backported in [#101360](https://github.com/ClickHouse/ClickHouse/issues/101360): Fix `MAX`/`MIN` aggregate functions on `Decimal` columns returning incorrect results when JIT compilation is enabled (after the compilation threshold is reached).
