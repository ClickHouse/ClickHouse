# 2026 Changelog

### ClickHouse release v26.3.5.12-lts (172e4255c5c) FIXME as compared to v26.3.4.11-lts (8186e0e780d)

#### Performance Improvement
* Backported in [#101736](https://github.com/ClickHouse/ClickHouse/issues/101736): Fix significant INSERT performance regression when `deduplicate_insert=enable` (default since 26.2) by deferring data hash computation from squashing to the sink and using batch column hashing via `updateHashWithValueRange`, reducing overhead from ~2.5s to ~0.5s for 5M rows with 22 columns. [#101494](https://github.com/ClickHouse/ClickHouse/pull/101494) ([Sema Checherinda](https://github.com/CheSema)).

#### Improvement
* Backported in [#101949](https://github.com/ClickHouse/ClickHouse/issues/101949): Optimize cache loading on server startup. [#101500](https://github.com/ClickHouse/ClickHouse/pull/101500) ([Kseniia Sumarokova](https://github.com/kssenii)).
* Backported in [#102140](https://github.com/ClickHouse/ClickHouse/issues/102140): Bumped internal mongo-c-driver from 2.2.2 to 2.2.3. [#101655](https://github.com/ClickHouse/ClickHouse/pull/101655) ([Robert Schulze](https://github.com/rschu1ze)).

#### Bug Fix (user-visible misbehavior in an official stable release)
* Backported in [#101962](https://github.com/ClickHouse/ClickHouse/issues/101962): Fix wrong results when JOIN with shard-by-PK optimization uses query condition cache and some parts are filtered out by cached conditions. [#100926](https://github.com/ClickHouse/ClickHouse/pull/100926) ([Groene AI](https://github.com/groeneai)).
* Backported in [#101987](https://github.com/ClickHouse/ClickHouse/issues/101987): Fixed "Target table doesn't exist" errors for materialized views with inner tables during async startup, caused by incorrect startup dependency ordering. [#100946](https://github.com/ClickHouse/ClickHouse/pull/100946) ([Nikolay Degterinsky](https://github.com/evillique)).
* Backported in [#101563](https://github.com/ClickHouse/ClickHouse/issues/101563): Fix insert-select from delta lake cluster with replicated merge tree. [#101299](https://github.com/ClickHouse/ClickHouse/pull/101299) ([Konstantin Vedernikov](https://github.com/scanhex12)).
* Backported in [#102076](https://github.com/ClickHouse/ClickHouse/issues/102076): Fix use-after-scope in parallel deserialization of `Object` type dynamic paths, which could cause crashes when reading tables with many dynamic paths. [#101823](https://github.com/ClickHouse/ClickHouse/pull/101823) ([Antonio Andelic](https://github.com/antonio2368)).
* Backported in [#102102](https://github.com/ClickHouse/ClickHouse/issues/102102): Fixed incorrect output in function `formatDateTime` with formatter`%W` with certain (non-default) formatting settings. [#101847](https://github.com/ClickHouse/ClickHouse/pull/101847) ([Robert Schulze](https://github.com/rschu1ze)).
* Backported in [#102030](https://github.com/ClickHouse/ClickHouse/issues/102030): Fix UDF registry loss when ZooKeeper session expires during periodic refresh — all user-defined functions could become unavailable until a full refresh succeeds. [#101891](https://github.com/ClickHouse/ClickHouse/pull/101891) ([Nikita Fomichev](https://github.com/fm4v)).
* Backported in [#102094](https://github.com/ClickHouse/ClickHouse/issues/102094): The `splitByString` tokenizer now rejects empty separator strings. [#101928](https://github.com/ClickHouse/ClickHouse/pull/101928) ([Robert Schulze](https://github.com/rschu1ze)).

#### NOT FOR CHANGELOG / INSIGNIFICANT

* Backported in [#102088](https://github.com/ClickHouse/ClickHouse/issues/102088): Fix heap-buffer-overflow in `leftPad/rightPad` functions. [#101951](https://github.com/ClickHouse/ClickHouse/pull/101951) ([Robert Schulze](https://github.com/rschu1ze)).
