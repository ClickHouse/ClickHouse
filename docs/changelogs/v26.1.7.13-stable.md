# 2026 Changelog

### ClickHouse release v26.1.7.13-stable (c5bf58bd2ff) FIXME as compared to v26.1.6.6-stable (078d5649b09)

#### Improvement
* Backported in [#100464](https://github.com/ClickHouse/ClickHouse/issues/100464): Set `this->exceptions(std::ios::badbit);` for the std's streams inside poco. [#96455](https://github.com/ClickHouse/ClickHouse/pull/96455) ([Sema Checherinda](https://github.com/CheSema)).

#### Bug Fix (user-visible misbehavior in an official stable release)
* Backported in [#99158](https://github.com/ClickHouse/ClickHouse/issues/99158): Disallow dropping column when its subcolumns are used in other columns default/alias expressions and use analyzer for default expressions on alter drop column. [#98569](https://github.com/ClickHouse/ClickHouse/pull/98569) ([Nikita Mikhaylov](https://github.com/nikitamikhaylov)).
* Backported in [#100080](https://github.com/ClickHouse/ClickHouse/issues/100080): Fix ignoring of TABLE_UUID_MISMATCH for non analyzer. [#99380](https://github.com/ClickHouse/ClickHouse/pull/99380) ([Azat Khuzhin](https://github.com/azat)).
* Backported in [#100356](https://github.com/ClickHouse/ClickHouse/issues/100356): Fix incorrect partition pruning for `toWeek()` that caused queries with `WHERE toWeek(date, mode) = N` to return empty results for weeks 49-52 on tables partitioned by `toYYYYMM(date)`. [#99542](https://github.com/ClickHouse/ClickHouse/pull/99542) ([Takumi Hara](https://github.com/takumihara)).
* Backported in [#100168](https://github.com/ClickHouse/ClickHouse/issues/100168): ... [#100024](https://github.com/ClickHouse/ClickHouse/pull/100024) ([Shaohua Wang](https://github.com/tiandiwonder)).
* Backported in [#100265](https://github.com/ClickHouse/ClickHouse/issues/100265): Fixed a rare crash in case of throwing a "Memory Limit" exception during building the text index. [#100213](https://github.com/ClickHouse/ClickHouse/pull/100213) ([Anton Popov](https://github.com/CurtizJ)).
* Backported in [#100342](https://github.com/ClickHouse/ClickHouse/issues/100342): Fixed a `LOGICAL_ERROR` exception in queries involving `Dynamic` columns with cross joins and runtime filters, caused by `ColumnVariant::filter` sharing variant column pointers instead of cloning them in the `hasOnlyNulls` optimization path. Closes https://github.com/ClickHouse/ClickHouse/pull/100147. [#100234](https://github.com/ClickHouse/ClickHouse/pull/100234) ([Pavel Kruglov](https://github.com/Avogar)).
* Backported in [#100594](https://github.com/ClickHouse/ClickHouse/issues/100594): Fix LOGICAL_ERROR exception "Stream ... not found" when inserting into a table with nested `Array(JSON)` columns in wide parts with `optimize_on_insert=0`. [#100475](https://github.com/ClickHouse/ClickHouse/pull/100475) ([Pavel Kruglov](https://github.com/Avogar)).
* Backported in [#100887](https://github.com/ClickHouse/ClickHouse/issues/100887): Validate file entry paths in backup metadata to reject path traversal, absolute paths, and empty names during `RESTORE`. [#100483](https://github.com/ClickHouse/ClickHouse/pull/100483) ([Pablo Marcos](https://github.com/pamarcos)).
* Backported in [#100725](https://github.com/ClickHouse/ClickHouse/issues/100725): Fixes NOT_FOUND_COLUMN_IN_BLOCK cases where the projection SELECT part has columns that do not exist in the original SELECT part of the query. Closes [#100194](https://github.com/ClickHouse/ClickHouse/issues/100194). [#100623](https://github.com/ClickHouse/ClickHouse/pull/100623) ([Yarik Briukhovetskyi](https://github.com/yariks5s)).
* Backported in [#100918](https://github.com/ClickHouse/ClickHouse/issues/100918): Fix server crash when using `has()` function with PREWHERE/WHERE on a Tuple key containing LowCardinality elements. [#100760](https://github.com/ClickHouse/ClickHouse/pull/100760) ([Groene AI](https://github.com/groeneai)).
* Backported in [#100868](https://github.com/ClickHouse/ClickHouse/issues/100868): Fix `If`, `Distinct`, `DistinctIf`, `IfState ` aggregate function combinators with `Tuple` return type and one or more `Nullable` argument not being able to read older serialized states after introduction of `Nullable(Tuple)`. Closes [#98917](https://github.com/ClickHouse/ClickHouse/issues/98917). [#100826](https://github.com/ClickHouse/ClickHouse/pull/100826) ([Nihal Z. Miaji](https://github.com/nihalzp)).
