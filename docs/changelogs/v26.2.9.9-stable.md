# 2026 Changelog

### ClickHouse release v26.2.9.9-stable (94b91bf8dcf) FIXME as compared to v26.2.8.15-stable (5f35e27d1a1)

#### Performance Improvement
* Backported in [#101914](https://github.com/ClickHouse/ClickHouse/issues/101914): Fix significant INSERT performance regression when `deduplicate_insert=enable` (default since 26.2) by deferring data hash computation from squashing to the sink and using batch column hashing via `updateHashWithValueRange`, reducing overhead from ~2.5s to ~0.5s for 5M rows with 22 columns. [#101494](https://github.com/ClickHouse/ClickHouse/pull/101494) ([Sema Checherinda](https://github.com/CheSema)).

#### Improvement
* Backported in [#101601](https://github.com/ClickHouse/ClickHouse/issues/101601): Cgroupv2 memory tracking now excludes `slab_reclaimable` from kernel memory, giving a more accurate measure of non-reclaimable memory usage. [#100901](https://github.com/ClickHouse/ClickHouse/pull/100901) ([Antonio Andelic](https://github.com/antonio2368)).

#### Bug Fix (user-visible misbehavior in an official stable release)
* Backported in [#101985](https://github.com/ClickHouse/ClickHouse/issues/101985): Fixed "Target table doesn't exist" errors for materialized views with inner tables during async startup, caused by incorrect startup dependency ordering. [#100946](https://github.com/ClickHouse/ClickHouse/pull/100946) ([Nikolay Degterinsky](https://github.com/evillique)).
* Backported in [#102073](https://github.com/ClickHouse/ClickHouse/issues/102073): Fix use-after-scope in parallel deserialization of `Object` type dynamic paths, which could cause crashes when reading tables with many dynamic paths. [#101823](https://github.com/ClickHouse/ClickHouse/pull/101823) ([Antonio Andelic](https://github.com/antonio2368)).
* Backported in [#102100](https://github.com/ClickHouse/ClickHouse/issues/102100): Fixed incorrect output in function `formatDateTime` with formatter`%W` with certain (non-default) formatting settings. [#101847](https://github.com/ClickHouse/ClickHouse/pull/101847) ([Robert Schulze](https://github.com/rschu1ze)).
* Backported in [#102028](https://github.com/ClickHouse/ClickHouse/issues/102028): Fix UDF registry loss when ZooKeeper session expires during periodic refresh — all user-defined functions could become unavailable until a full refresh succeeds. [#101891](https://github.com/ClickHouse/ClickHouse/pull/101891) ([Nikita Fomichev](https://github.com/fm4v)).
* Backported in [#102092](https://github.com/ClickHouse/ClickHouse/issues/102092): The `splitByString` tokenizer now rejects empty separator strings. [#101928](https://github.com/ClickHouse/ClickHouse/pull/101928) ([Robert Schulze](https://github.com/rschu1ze)).

#### NOT FOR CHANGELOG / INSIGNIFICANT

* Backported in [#102086](https://github.com/ClickHouse/ClickHouse/issues/102086): Fix heap-buffer-overflow in `leftPad/rightPad` functions. [#101951](https://github.com/ClickHouse/ClickHouse/pull/101951) ([Robert Schulze](https://github.com/rschu1ze)).
